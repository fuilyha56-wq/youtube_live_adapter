# YouTube Live Adapter 出入站流程补全 — 适配 chatter/adapter 生态

> Created: 2026-06-13

## 背景

对比 bilibili_live_adapter 的完整出入站流程，YouTube adapter 的 dispatcher 在构建 MessageEnvelope 时缺少若干关键字段，导致下游组件（anima_chatter、default_chatter 等）无法正确消费 YouTube 消息。

框架消息流转链路：
```
Dispatcher → envelope → MessageConverter.envelope_to_message → Message → stream → chatter
```

关键映射规则（来自 MessageConverter 源码）：
- `envelope["raw_message"]` → `Message.raw_data`
- `message_info.extra` → `Message.extra`（通过 `**extra_data` 展开）
- `message_info.additional_config` → 不自动映射到 Message，但标准做法仍应写入

## 改动清单

### 1. dispatcher.py — 每个 `_build_*_envelope` 方法补全 envelope 字段

**问题**：YouTube dispatcher 构建了 `additional` dict 但没有写入 `envelope["message_info"]["additional_config"]`，也没有写入 `envelope["raw_message"]`。

**改动**：在所有 `_build_*_envelope` 方法中，`_inject_source_into_extra` 调用之后，添加：

```python
envelope["message_info"]["additional_config"] = additional
envelope["raw_message"] = renderer
```

涉及方法（共9个）：
- `_build_text_envelope`
- `_build_super_chat_envelope`
- `_build_super_sticker_envelope`
- `_build_gift_purchase_envelope`
- `_build_gift_redemption_envelope`
- `_build_gift_announcement_envelope`
- `_build_milestone_envelope`
- `_build_paid_sticker_envelope`
- `_build_paid_message_envelope`

### 2. dispatcher.py — `_apply_user` 添加 `user_avatar` 参数

**问题**：bilibili dispatcher 通过 `_apply_user` 注入 `user_avatar=uface`，YouTube 缺失。

**改动**：
1. `_apply_user` 方法签名添加 `user_avatar: str = ""` 参数
2. 在 `builder.from_user(...)` 调用中传入 `user_avatar=user_avatar`
3. 各 `_build_*_envelope` 方法调用 `_apply_user` 时传入 `user_avatar=str(renderer.get("authorPhotoUrl", ""))`

### 3. dispatcher.py — 添加 info 级别消息日志

**问题**：bilibili dispatcher 每条消息都有 `logger.info` 级别日志，YouTube 只有 debug。

**改动**：在每个 `_build_*_envelope` 方法的 return 前添加 `logger.info` 日志，格式参考 bilibili：
- 普通弹幕：`收到弹幕 [room={video_id}] {nickname}({user_id[:8]}...) 说: {text}`
- SC：`收到SC [room={video_id}] {nickname}({user_id[:8]}...) ¥{amount}: {text}`
- 付费消息：`收到付费消息 [room={video_id}] {nickname}({user_id[:8]}...) {amount_display}: {text}`
- 其他类型类似

### 4. dispatcher.py — 各 `_build_*_envelope` 方法补充 `event_type` 字段

**问题**：bilibili 的 `additional_config` 包含 `event_type` 字段（"danmaku"/"gift"/"super_chat"/"guard"），YouTube 缺失。

**改动**：在各 `_build_*_envelope` 方法中，向 `additional` dict 添加 `event_type` 字段：
- `_build_text_envelope` → `"event_type": "danmaku"`
- `_build_super_chat_envelope` → `"event_type": "super_chat"`
- `_build_super_sticker_envelope` → `"event_type": "super_sticker"`
- `_build_gift_purchase_envelope` → `"event_type": "gift_purchase"`
- `_build_gift_redemption_envelope` → `"event_type": "gift_redemption"`
- `_build_gift_announcement_envelope` → `"event_type": "gift_announcement"`
- `_build_milestone_envelope` → `"event_type": "milestone"`
- `_build_paid_sticker_envelope` → `"event_type": "paid_sticker"`
- `_build_paid_message_envelope` → `"event_type": "paid_message"`

### 5. dispatcher.py — 更新模块文档字符串

**改动**：在模块顶部文档中补充说明 envelope 字段规范，包括 `additional_config`、`raw_message`、`user_avatar` 等。

## 不需要改动的文件

- `src/api.py` — 已完成
- `src/client.py` — 已完成
- `src/sender.py` — 出站发送器无需改动
- `src/currency.py` — 货币转换无需改动
- `config.py` — 配置无需改动
- `plugin.py` — 已完成 system_reminder、连续失败计数、出站日志

## 验收标准

1. 每个 `_build_*_envelope` 方法都写入 `envelope["message_info"]["additional_config"]` 和 `envelope["raw_message"]`
2. `_apply_user` 支持 `user_avatar` 参数，且各构建方法传入 renderer 中的 `authorPhotoUrl`
3. 每个消息构建方法都有 `logger.info` 级别日志
4. `additional_config` 包含 `event_type` 和 `original_type` 字段
5. 所有改动通过 ruff 和 pyright 静态检查
6. anima_chatter 能通过 `msg.extra.get("source_platform")` 正确识别 `"youtube_live"` 来源
7. `Message.raw_data` 不再为 None（包含原始 renderer dict）
